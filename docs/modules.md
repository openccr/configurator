# Module System

## Types (`types/index.ts`)

```typescript
type BusType = 'I2C' | 'SPI' | 'UART' | 'NEOPIXEL' | 'MODBUS' | 'GPIO' | 'ADC' | 'SOLENOID'
type PortDirection = 'in' | 'out' | 'bidir'

interface PortDef {
  id: string          // unique within module, used in Wire
  label: string       // displayed in card
  busType: BusType
  direction: PortDirection
  multiDrop: boolean  // true = multiple wires allowed (I2C, GPIO pool)
  rightSide?: boolean // true = dot on right edge; default false (left)
  implicit?: boolean  // true = consumed by MCU GPIO pool, never rendered
}

interface ModuleDefinition {
  type: ModuleType
  label: string
  toolbarLabel?: string   // shorter label for toolbar button; falls back to label
  description: string
  colorGroup: 'sensor' | 'actuator' | 'display' | 'storage' | 'control' | 'core'
  ports: PortDef[]        // MCU has empty array (ports generated dynamically)
}
```

## MCU Dynamic Ports

MCU ports are generated by `getMcuPorts(busSlots)` from `portUtils.ts`. Port IDs:
- `'GPIO'` — multiDrop GPIO pool, right side
- `'MODBUS'` — bidir, left side
- `'I2C-1'`, `'I2C-2'`, … — expandable, right side, multiDrop
- `'SPI-1'`, `'SPI-2'`, … — expandable, right side, single-drop
- `'UART-1'`, … — expandable, right side, single-drop
- `'NEOPIXEL-1'`, … — expandable, right side, single-drop

Expandable buses: user clicks `+` on last slot (when wired) to add, `−` (when unwired) to remove. Min 1 slot each. Default: 1 slot per bus.

## GPIO Accounting (MCU Card)

MCU displays `GPIO (N)` where N = explicit wires to GPIO pool + implicit pins from all reachable peripherals. Implicit pins counted by `def.ports.filter(p => p.implicit).length` for each reachable module (BFS from MCU through all wires).
